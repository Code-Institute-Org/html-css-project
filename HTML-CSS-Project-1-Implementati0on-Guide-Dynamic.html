Project Implementation Guide

Phase 1: Ideation & Initial Setup (4 hours)

Description: In this phase, the primary goal is to lay a solid foundation for the project by defining its purpose, audience, and design. This involves brainstorming ideas, creating user stories, and setting up the project environment.

Ideation and User Story Definition (1.5 hours)

This task involves brainstorming and refining project ideas, defining user stories that capture the project's essential features and functionality, and documenting these stories in the README. This process helps clarify the project's goals and ensures that the development is user-centered.

Tasks:

  • Brainstorm and refine project ideas.
  • Define and prioritize user stories and their acceptance criteria (Must, Should, Could).
  • Add the project definition, user stories, and acceptance criteria to the README (User Stories section).

Pass Guidance

Associated Criteria:
  • LO1.1: Design a website that incorporates a main navigation menu and a structured layout.
  • LO1.3: Design the organization of information on the page following the principles of user experience design.
  • LO4.1: Write a README.md file for the web application that explains its purpose, the value that it provides to its users, and the deployment procedure.
Justification:

LO1.1: Designing a main navigation menu and structured layout is essential in the early planning phase to ensure the website has a clear and functional structure from the outset.

LO1.3: Organizing information following user experience principles is crucial during the ideation and planning phase to ensure the website will be user-friendly.

LO4.1: Documenting user stories in the README aligns with the need for clear documentation and understanding of the project's purpose and value.

Pass Grade: Expected Pass Performance (Do's):
  • Think about the specific needs of your target audience.
  • Use user stories and their acceptance criteria to guide the development process.
  • Include user stories and their acceptance criteria in the README.
Pass Grade Areas for Improvement (Don'ts):
  • Skip this step; clear planning is crucial for success.
  • Overlook the importance of user stories in guiding development.

Initial Planning and Wireframing (1.5 hours)

Learners will outline the website's structure and design through wireframes. This includes sketching key pages, choosing color schemes and fonts, and planning content placement. Documenting these decisions helps in creating a clear design blueprint and facilitates a user-centric approach.

Tasks:

  • Sketch wireframes for each key page (or section for single-page applications) using Balsamiq.
  • Ensure wireframes include intuitive and strategically placed navigation options.
  • Ensure wireframes reflect the Must and Should user stories.
  • Choose color schemes and fonts using tools like color contrast checkers to meet accessibility standards.
  • Briefly document design decisions in the README.
  • Add your wireframes to the README.

Pass Guidance

Associated Criteria:
  • LO1.2: Design a website that meets accessibility guidelines.
  • LO1.3: Design the organization of information on the page following the principles of user experience design.
  • LO1.4: Ensure that foreground information is never distracted by backgrounds.
  • LO4.1: Write a README.md file for the web application that explains its purpose, the value that it provides to its users, and the deployment procedure.
Justification:

LO1.2: Ensuring the website meets accessibility guidelines is fundamental for inclusivity and usability.

LO1.3: Organizing information according to user experience design principles ensures the website will be user-friendly and intuitive.

LO1.4: Maintaining clear foreground information ensures content is easily comprehensible and not overshadowed by background elements.

LO4.1: Documenting design decisions in the README is essential for future maintainability and understanding the design rationale.

Pass Grade: Expected Pass Performance (Do's):
  • Create wireframes for each key page (or section in creating a single page application) using Balsamiq. (LO1.3)
  • Prioritize and structure information clearly.
  • Ensure that the wireframes include intuitive and strategically placed navigation options (LO1.3)
  • Ensure an intuitive user experience.
  • Ensure that the wireframes reflect the Must and Should user stories (LO1.3)
  • Ensure chosen color schemes meet accessibility standards. (LO1.2)
  • Ensure good contrast throughout allowing content to take center stage on the screen.
  • Select color schemes that complement the layout.
  • Ensure that foreground information is never distracted by backgrounds. (LO1.4)
  • Maintain an information hierarchy supported by consistent imagery and text.
  • Keep designs simple and elegant.
  • Briefly document your wireframe, color, and layout decisions in the README. (LO4.1)
  • Add your wireframes to the README. (LO4.1)
Pass Grade Areas for Improvement (Don'ts):
  • Start coding without a clear design plan.
  • Neglect accessibility standards when choosing colors and fonts.
  • Create wireframes that result in difficulty in Navigation from one page, or if using a single page application, one section to another.
  • Avoid images and text overlays with poor contrast ratios that are incomprehensible.
  • Do not omit UX design write-up or wireframes.
  • Avoid presenting information without considering categorization priority.
  • Avoid varying information across different pages, resolutions, or devices.
  • Ensure content is not difficult to understand against the background.
  • Do not miss relevant steps or include them without detailed descriptions.

Environment Setup and Version Control (1 hour)

Setting up the development environment includes creating a GitHub repository, organizing project files, and linking necessary resources. Proper documentation of this setup ensures that the project is well-organized and version-controlled from the start.

Tasks:

  • Set up a GitHub repository.
  • Create a basic project structure with necessary folders (assets, CSS, images).
  • Link CSS and any external resources like Bootstrap, Google Fonts, and Font Awesome.
  • Document the setup process in the README (Setup and Installation section).

Pass Guidance

Associated Criteria:
  • LO3.2: Use Git & GitHub for version control of an interactive web application up to deployment.
  • LO4.6: Place CSS code in external files linked to the HTML page in the HEAD element.
  • LO4.9: Group files in directories by file type.
  • LO5.1: Use consistent and effective markdown formatting to produce a README file in English that is well-structured, easy to follow, and has few grammatical errors.
Justification:

LO3.2: Using Git & GitHub from the start ensures proper version control.

LO4.6: Linking CSS in external files is a best practice for maintainability.

LO4.9: Organizing files by type from the beginning is crucial for project organization.LO5.1: Clear, consistent documentation is essential for project maintainability and clarity.

Pass Grade: Expected Pass Performance (Do's):
  • Commit regularly to GitHub with meaningful commit messages. (LO3.2)
  • Organize files logically from the start. (LO4.9)
  • Link CSS in external files placed appropriately in the HEAD element. (LO4.6)
  • Use consistent and effective markdown formatting to produce a README file in English that is well-structured, easy to follow, and has few grammatical errors. (LO5.1)
Pass Grade Areas for Improvement (Don'ts):
  • Delay setting up version control.
  • Make a few big commits instead of consistent regular commits. (LO3.2)
  • Use CSS ineffectively by relying too much on in-line styles. (LO4.6)
  • Ignore directory structure conventions and place files haphazardly. (LO4.9)
  • Forget to test the setup to ensure everything is working correctly.
  • Produce a README file with inconsistent formatting or many grammatical errors. (LO5.1)

Phase 2: Must User Stories Implementation & Testing (14 hours)

Description: This phase focuses on implementing the core functionality of the website as outlined in the Must user stories. It involves coding essential features, validating code, and ensuring visual and functional alignment with initial plans.

Must User Stories Implementation (11 hours)

Essential features are developed based on Must user stories. This task ensures that the core functionality is in place, and the website meets the initial design requirements. Validating HTML and CSS code ensures that the implementation is technically sound.

Tasks:

  • Implement essential features defined in Must user stories.
  • Ensure the website structure is functional and meets initial design requirements.
  • Validate HTML and CSS code using W3C and Jigsaw validators.
  • Document key coding decisions and any challenges faced in the README (Features section).

Pass Guidance

Associated Criteria:
  • LO1.1: Design a website that incorporates a main navigation menu and a structured layout.
  • LO1.2: Design a website that meets accessibility guidelines.
  • LO1.4: Ensure that foreground information is never distracted by backgrounds.
  • LO1.5: Include graphics that are consistent in style and color.
  • LO1.6: Design the site to allow the user to initiate and control actions such as the playing of audio/video.
  • LO2.2: Write custom HTML code that passes through the official W3C validator with no issues.
  • LO2.3: Write custom CSS code that passes through the official Jigsaw validator with no issues.
  • LO2.4: Incorporate images that are of sufficient resolution to not appear pixelated or stretched.
  • LO3.2: Use Git & GitHub for version control of an interactive web application up to deployment.
Justification:

LO1.1: Ensuring the navigation menu and structured layout are functional aligns with the need for clear and intuitive site navigation.

LO1.2: Meeting accessibility guidelines ensures the website is usable by all, including those with disabilities.

LO1.4: Keeping foreground information undistracted by backgrounds is crucial for maintaining focus on core content.

LO1.5: Consistent graphics enhance the visual coherence and professionalism of the website.

LO1.6: Allowing user control over actions like playing audio/video ensures a better user experience.

LO2.2: Validating HTML code through W3C ensures compliance with web standards.

LO2.3: Validating CSS code through Jigsaw ensures it meets industry standards.

LO2.4: Incorporating high-resolution images ensures the visual quality of the website is maintained without pixelation or stretching.

LO3.2: Using Git & GitHub for version control is essential for maintaining a structured development process, allowing for tracking changes, and collaboration, and ensuring an organized and recoverable project history.

Pass Grade: Expected Pass Performance (Do's):
  • Focus on implementing the most critical features first. (LO1.1, LO1.4)
  • Place navigation elements strategically for ease of use. (LO1.1)
  • Use good contrast throughout allowing content to take center stage on the screen. (LO1.2)
  • Ensure that the foreground information is not distracted by backgrounds. (LO1.4)
  • Use consistent graphics in size, resolution, and layout. (LO1.5)
  • Ensure audio/video control options are user-initiated. (LO1.6)
  • Regularly validate your HTML code to catch errors early. (LO2.2)
  • Regularly validate your CSS code to catch errors early. (LO2.3)
  • Incorporate high-resolution images to avoid pixelation or stretching. (LO2.4)
  • Commit regularly to GitHub with meaningful commit messages. (LO3.2)
Pass Grade Areas for Improvement (Don'ts):
  • Avoid having broken navigation links. (LO1.1)
  • Do not create difficulty in navigating from one page to another. (LO1.1)
  • Avoid images and text overlays with poor contrast ratios. (LO1.2)
  • Do not vary information presentation across different pages, resolutions, or devices. (LO1.4)
  • Avoid inconsistent graphics color in the layout. (LO1.5)
  • Ensure pop-ups or videos are not obtrusive and do not autoplay on mute. (LO1.6)
  • Do not neglect HTML validation; ensure it passes W3C checks. (LO2.2)
  • Do not neglect CSS validation; ensure it passes Jigsaw checks. (LO2.3)
  • Do not use images that appear stretched or pixelated. (LO2.4)
  • Make few large commits instead of consistent regular commits. (LO3.2)

Visual Progress & Testing (1 hour)

The website is tested on different devices to ensure visual and functional consistency. Adjustments are made based on initial tests, and changes are documented. This phase ensures that the website's appearance and functionality align with the design specifications.

Tasks:

  • Preview and test the website on different (virtual) devices using Chrome Dev Tools and Lighthouse.
  • Adjust styles as needed based on initial tests.
  • Briefly document any major changes made during testing in the README (Testing section).

Pass Guidance

Associated Criteria:
  • LO2.6: Use CSS media queries across the application to ensure the layout changes appropriately and maintains the page's structural integrity across device screen sizes.
Justification:

LO2.6: Using CSS media queries is essential to ensure that a website is responsive and provides a consistent user experience across different devices and screen sizes. This involves adjusting the layout, images, and other elements to fit the screen properly, enhancing usability and accessibility.

Pass Grade: Expected Pass Performance (Do's):
  • Ensure the website layout adapts seamlessly to different screen (virtual) sizes, such as desktops, tablets, and mobile devices using Chrome Developer Tools and Lighthouse. (LO2.6)
  • Use media queries to adjust font sizes, images, and other elements for better readability and accessibility on various devices. (LO2.6)
  • Make any adjustments to ensure consistency. (LO2.6)
Pass Grade Areas for Improvement (Don'ts):
  • Skip testing; it’s crucial for finding issues.
  • Ignore feedback from testing.

Merit Guidance

Associated Criteria:
  • LO2.1: Document any bugs found and their fixes and explanation of any bugs that are left unfixed.
Justification:

LO2.1: Documenting bugs and their resolutions is essential for maintaining the quality and reliability of the website. It ensures transparency and facilitates future debugging, even with limited time allocated for documentation.

Merit Grade: Expected Performance (Do's):
  • Briefly document any significant bugs found and their fixes, along with explanations for any critical bugs that remain unfixed. (LO2.1)
Merit Grade Areas for Improvement (Don'ts):
  • Do not omit documentation of major testing steps or significant bugs. (LO2.1)

Distinction Guidance

Justification:

Robust Code: Thorough testing ensures reliability across different environments.

Compliant Code: Documenting test cases and results provides evidence of comprehensive testing and resolution of issues.

Confirmation: Adjustments based on feedback improve usability and user satisfaction.

Consistency: A polished, consistent website provides a professional user experience.

Compliant Code: Passing validation checks ensures adherence to web standards.

Distinction Grade: Expected Performance (Do's):
  • Conduct thorough testing on multiple devices and browsers to ensure consistent performance. (Robust Code)
  • Document all test cases and results, highlighting any issues found and how they were resolved. (Compliant Code)
  • Make necessary adjustments based on feedback from initial tests to improve usability. (Confirmation)
  • Ensure the website is highly polished with no visual or functional inconsistencies. (Consistency)
  • Code passes through official validators (W3C and Jigsaw) with no issues. (Compliant Code)
Distinction Grade Areas for Improvement (Don'ts):
  • Skip testing on a variety of devices and browsers. (Robust Code)
  • Ignore or fail to document testing results. (Compliant Code)
  • Leave identified issues unresolved. (Confirmation)
  • Have visual or functional inconsistencies after testing. (Consistency)
  • Code does not pass validation checks. (Compliant Code)
